User Routes
# Route method Endpoint Params Description
1 Signup POST /user/signup {"email": "user@example.com", "username": "abcde", password: "123456"} For user registration on the platform
2 Login POST /user/login {"email": "user@example.com", password: "123456"} Sign into user account
3 Recover Password POST /user/recoverpassword {"email": "user@example.com"} For user's account password recovery; OTP will be sent to email
4 Reset Password POST /user/resetpassword {"email": "user@example.com", "code": "12345678", "new_password": "123456"} Reset password. Use OTP sent to email from /recoverpassword
5 Update Profile PATCH /user/updateprofile/{user_id} {"full_name": "Jack Robinson", "birthday": "12-12-1912", "city": "awka", "sex": "male", "bio": "i code", "image_url": "url_or_path_to_profilepic"} Update user's profile. Required params: [full_name, birthday, city, sex]
6 Follow other user(s) PATCH /user/follow/{user_id} {"follow_list": ["111111111111", "222222222222222"]} Follow other users on the platform
7 User Profile GET /user/{user_id} null Fetch user's data


Hype Routes
# Route method Endpoint Params Description
1 Make a Hype POST /hype {"user_id": "1212121231121231", "hype": "A hype body example here", img_urls: ["path_or_url_to_image1", "path_or_url_to_image2"]} To make a hype; img_urls is optional
2 Edit a Hype PATCH /hype/edit/{hype_id} {"hype": "A hype body example being edited here", img_urls: ["path_or_url_to_image1", "path_or_url_to_image2"]} To edit a hype
3 Get all Hypes for a user GET /hype/user/{user_id} null Fetches all hypes made by user
